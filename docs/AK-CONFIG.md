# AK 配置说明

## AK 是什么

AK（Access Key）是 1688 AI 版的身份认证凭证，类似于"账号密码"，用于：
- 验证你的身份
- 调用 1688 平台的接口
- 访问你绑定的店铺信息

**安全提示**：
- AK 仅存储在你的本地电脑
- 每次请求都会进行加密传输
- 建议妥善保管，不要分享给他人

## 如何获取 AK

### 步骤 1：下载 1688 AI 版 APP

- **安卓**：应用商店搜索「1688 AI版」
- **iOS**：App Store 搜索「1688 AI版」

或访问：https://air.1688.com/kapp/1688-ai-app/pages/home

### 步骤 2：获取 AK

1. 打开 1688 AI 版 APP
2. 使用 1688 账号登录
3. 进入「我的」→「开发者设置」
4. 复制 AK（一串字母数字组合，通常40+位）

### 步骤 3：配置 AK

提供三种配置方式：

---

## 方式一：对话中直接提供（推荐）

**直接在对话中告诉 Agent**：

```
我的AK是 xxxx
```

Agent 会自动：
1. 识别 AK 格式
2. 通过 `configure.py` 保存到 `~/.openclaw/openclaw.json`（持久化）
3. 在当前会话的后续命令中手动注入 AK 立即使用
4. 重启 Gateway 后所有新会话由框架自动注入

---

## 方式二：命令行配置

如果你熟悉命令行，可以使用提供的配置工具：

```bash
cd {baseDir}/scripts && python3 configure.py your_ak_here
```

此工具会：
- 验证 AK 格式
- 写入 `~/.openclaw/openclaw.json`（`skills.entries.1688-skills.env.ALI_1688_AK`）
- 重启 Gateway 后自动生效

---

## 方式三：手动配置

**临时配置（仅当前终端会话有效）**：
```bash
export ALI_1688_AK=your_ak_here
```

**永久配置（推荐写入配置文件）**：
```bash
# 添加到 ~/.zshrc（Zsh 用户）或 ~/.bashrc（Bash 用户）
echo 'export ALI_1688_AK=your_ak_here' >> ~/.zshrc

# 使配置生效
source ~/.zshrc
```

---

## 验证配置

配置完成后，可以通过以下方式验证：

### 方法 1：通过 Agent 验证

告诉 Agent："查看我绑定的店铺"

如果 AK 配置正确，Agent 会返回店铺列表；否则会提示配置 AK。

### 方法 2：检查环境变量

```bash
echo $ALI_1688_AK
```

应输出你的 AK（或至少显示前几位）。

---

## 常见问题

### Q: AK 配置后仍提示未配置

**可能原因**：
1. **未生效**：配置后需要在新终端窗口使用，或执行 `source ~/.zshrc`
2. **配置错误**：检查环境变量名是否为 `ALI_1688_AK`（注意大小写）
3. **格式问题**：确保没有多余空格或换行

**解决方法**：
```bash
# 检查当前值
echo $ALI_1688_AK

# 重新加载配置
source ~/.zshrc

# 或在新的终端窗口中重试
```

### Q: 如何更新 AK

**A**: 重新执行配置步骤即可，新配置会自动覆盖旧值。

### Q: AK 泄露了怎么办

**A**: 
1. 立即在 1688 AI 版 APP 中生成新的 AK（旧的会自动失效）
2. 使用新 AK 重新配置
3. 检查是否有异常操作

### Q: 多台电脑如何使用

**A**: 每台电脑需要单独配置 AK。由于 AK 与设备绑定，建议在每台设备上从 APP 获取独立的 AK。

### Q: AK 有效期多久

**A**: AK 本身长期有效，但建议定期（如每季度）更换以增强安全性。
